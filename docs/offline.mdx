---
title: "Offline Mode (Experimental)"
description: "Run Ante with local models - no API keys or internet required"
---

Ante can run entirely offline using local GGUF models via [llama.cpp](https://github.com/ggerganov/llama.cpp) (our current local inference engine). This means no API keys, no internet, and no data leaving your machine.

We expect to explore additional local engines over time, but the offline workflow and model format support will remain focused on a good "it just works" experience.

In parallel, we're building toward a truly self-contained agent stack; see our ongoing Rust effort at [AntigmaLabs/nanochat-rs](https://github.com/AntigmaLabs/nanochat-rs).

## How it works

Ante includes an integrated inference engine currently powered by llama.cpp. When you select offline mode, Ante:

1. Checks for llama.cpp installation (and offers to install/upgrade if needed)
2. Discovers GGUF models on your system
3. Detects running llama servers on local ports
4. Estimates memory requirements based on model size and context window
5. Runs inference locally through the engine

## Setting up

<Steps>
  <Step title="Launch Ante and open offline mode">
    Start Ante normally and use the offline mode selector in the TUI:

    ```bash
    ante
    ```
  </Step>
  <Step title="Install llama.cpp">
    If llama.cpp is not installed, Ante will prompt you to install it automatically. The engine is installed to `~/.ante/llama.cpp`.

    When a newer version is available, Ante will offer an upgrade option.
  </Step>
  <Step title="Select a model">
    Choose from:
    - **Verified models** — curated models tested for compatibility (downloaded automatically from Hugging Face)
    - **Local models** — GGUF files already on your system (auto-discovered)
    - **Running servers** — attach to an already-running llama server on a local port
  </Step>
  <Step title="Or use the CLI flag">
    ```bash
    ante --provider local "your prompt here"
    ```
  </Step>
</Steps>

## Model discovery

Ante automatically scans the following directories for GGUF model files:

| Directory | Description |
|---|---|
| `~/.ante/models` | Default model directory (configurable) |
| `~/.cache/llama.cpp` | llama.cpp cache |
| `~/.cache/huggingface/hub` | Hugging Face cache |
| `~/.llama/models` | Common llama model directory |

It handles:
- Single-file models (e.g., `model.gguf`)
- Sharded models (e.g., `Model-00001-of-00008.gguf`)
- Metadata extraction (context window, file size, KV cache info)

You can change the model directory in the TUI or by editing `~/.ante/offline-config.json`.

## Model preferences

You can configure per-model preferences in the TUI. These are saved automatically to `~/.ante/offline-config.json`:

| Setting | Description |
|---|---|
| `context_window` | Context window size (minimum 32K tokens) |
| `thinking` | Enable/disable chain-of-thought |
| `temperature` | Sampling temperature |

## Memory considerations

Ante estimates memory usage based on:

- **Model file size** — The base memory needed to load the model
- **KV cache** — Scales with context window size (bytes per token)
- **Shard count** — Multi-file models need proportional memory

<Tip>
For large models, reduce the context window to lower memory usage. The minimum is 32K tokens.
</Tip>

## Server management

Once a model is loaded, a local llama server runs in the background. Ante provides keyboard shortcuts to manage it:

| Shortcut | Action |
|---|---|
| `Ctrl+E` | Stop the currently connected server |
| `Ctrl+O` | View the server log |

When exiting Ante with a server running, you'll be prompted:
- **`s`** — Stop the server and exit
- **`k`** — Keep the server running and exit (prints PID for manual cleanup)
- **`Esc`** — Cancel and stay in Ante

You can also attach to a server that was kept running from a previous session.

## Verified models

Ante includes a curated list of verified models that are tested for compatibility and quality. These are shown prominently in the model selector and are downloaded automatically from Hugging Face when selected.

To add custom verified models, create `~/.ante/verified_models.json`. Your entries are merged with the built-in list (if a custom entry has the same filename as a built-in one, yours takes priority):

```json
{
  "models": [
    {
      "name": "My Custom Model",
      "repo": "username/repo-name",
      "filename": "model-Q4_K_M.gguf",
      "context_window": 32768,
      "file_size_mb": 5000,
      "kv_cache_bytes_per_token": 131072
    }
  ]
}
```

## Configuration reference

All offline mode configuration is stored in `~/.ante/offline-config.json`:

```json
{
  "version": "1.0.0",
  "model_directory": "~/.ante/models",
  "port": 8080,
  "last_model": "model-name",
  "model_preferences": {
    "model-id": {
      "model_id": "model-id",
      "context_window": 32768,
      "thinking_enabled": true,
      "temperature": 0.7
    }
  }
}
```

| Field | Description | Default |
|---|---|---|
| `model_directory` | Where to look for local GGUF models | `~/.ante/models` |
| `port` | Starting port for the llama server | `8080` |
| `last_model` | Last used model (auto-saved) | — |
| `model_preferences` | Per-model settings (context, thinking, temperature) | — |
